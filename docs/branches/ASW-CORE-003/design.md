# Plan Prompt for Agentic Software Organisation ASW-CORE-003

We need to add a new executive role: The VP Engineering. The VPE creates an execution plan based on the Vision PRD & architecture. It is the VP of Engineering's responsibility to determine the team's build-up strategy, as jumping directly from vision to final product is often inadvisable. The VP of Engineering's decisions will be vital for the Hiring Manager and the founder in selecting the appropriate team for the first phase. For example, phase one of the initial implementation may rely solely on local storage and in-memory databases to validate the product's core functionality. Subsequent phases could transition to a more robust production environment, perhaps utilizing separate containers with persistent storage, commercial-grade security layers, authentication, and optimized user experience. The VPE knows who they need for what phase. As part of this strategy, a list of generic role descriptions should be developed, which can be tailored at implementation to specific technologies and the intended product; these roles are typical of software development and support functions.  
To improve the organizational structure, several fundamental roles could be implemented, as exemplified below. These roles are optional and must be selected by the founder, who controls the budget, following recommendations from the VP of Engineering:

* **Project Manager:** Tasked with breaking work into distinct phases, this role requires detailed elaboration of each phase to set specific, testable targets that provide value to users. They also define the scope of changes as the company matures, such as determining when a version moves from internal use to production.  
* **Full Stack Test Specialist:** Concentrates on identifying the necessary testing frameworks and harnesses to ensure testing is conducted at the user acceptance level, rather than depending only on developer-led unit tests.  
* **Documentation Standards Lead:** Evaluates phase deliverables to identify needs for tutorials, training materials, and concept documents, enforcing standards for accuracy and logical flow. As the system evolves, they must ensure existing documentation reflects updates to user experience, system contracts, or behavior.  
* **Product Security Engineer:** While all developers should have security awareness, this role looks at the system holistically across information flow, privacy, and code. They review practices and standards to make recommendations regarding the implementation of design and phase requirements.  
* **DevOps Engineer:** Examines the functionality of the build and testing systems and how the product is deployed during any given phase. They may be experts in Docker, Docker Compose, or orchestration, performing general DevOps activities to manage how things run.  
* **Performance and Scaling Engineer:** Considers data volumes, scalability of services, and separation of concerns. Depending on product goals, certain components may require high performance; this engineer reviews the architecture, technology stack, and vision objectives to determine the best implementation approach.
